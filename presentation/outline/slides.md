# Slide Outline

## 1. Title
- 제목: OAK 카메라와 Gemma 4를 이용한 병원 장애물 인식 및 위치 추론
- 부제: 환자이송 로봇 적용을 위한 이미지 추론·좌표 후처리·거리 측정
- 핵심 메시지:
  - 병원 이미지에서 장애물 클래스를 분류하고 위치 좌표와 거리까지 연결한 흐름을 발표한다.

## 2. Used Technologies
- 제목: 문제 구조화 - 왜 병원 장애물 인식이 어려운가
- 핵심 메시지:
  - 단순 회피 수준의 AS-IS에서 의미 기반 대응이 가능한 TO-BE로 넘어가야 한다.
  - 병실/복도 환경은 비정형적이고 동적으로 변하므로 클래스와 위치를 함께 이해해야 한다.

## 3. Framework
- 제목: 전체 목표 프레임워크
- 핵심 메시지:
  - 입력 이미지가 의미 이해와 공간 정렬을 거쳐 행동 판단으로 이어지는 구조를 제시한다.
  - 환자이송 로봇이 실제 운영에서 쓸 수 있는 인지-공간-행동 프레임워크를 도식화한다.

## 4. Used Technologies
- 제목: 사용한 기술 설명
- 핵심 메시지:
  - OAK 카메라에서 RGB와 depth를 동시에 받는다.
  - Gemma 4 서버가 이미지 기반 장애물 클래스를 추론한다.
  - 후처리에서 모델 좌표를 로봇 사용 좌표로 변환한다.
  - C++ 테스트 하네스와 배치 실험으로 실제 동작을 검증했다.

## 5. Pipeline Diagram
- 제목: 파이프라인 다이어그램
- 핵심 메시지:
  - 입력 이미지에서 클래스, 좌표, 거리 정보까지 가는 전체 흐름을 한 장으로 설명한다.

## 6. Actual Run: Batch Image Inference
- 제목: 실제 동작 1 - 이미지 추론 결과
- 핵심 메시지:
  - 직접 수집한 병원 샘플 이미지들에 대해 장애물 클래스와 위치를 추론했다.
  - 클래스 빈도 분포를 통해 어떤 장애물이 반복적으로 등장하는지 확인했다.

## 7. Actual Run: Coordinate Post-processing
- 제목: 실제 동작 2 - 좌표 후처리와 병실 데모
- 핵심 메시지:
  - 모델의 정규화 좌표를 픽셀 좌표와 bottom-left 수학좌표로 변환했다.
  - 병실 샘플 이미지에서 실제 장애물 좌표를 시각화했다.

## 8. Actual Run: Real Depth Measurement
- 제목: 실제 동작 3 - 휠체어 거리 측정
- 핵심 메시지:
  - 실제 병원 복도 이미지에서 휠체어 좌표와 depth를 동시에 확인했다.
  - 후처리 좌표 `BL [451, 199]`, `TL [451, 521]`, depth `2827 mm`를 발표 사례로 사용한다.

## 9. Use Cases
- 제목: 어디에 쓰이면 좋은가
- 핵심 메시지:
  - 병실 출입, 복도 주행, 관제/재경로 등 환자이송 로봇 운영 시나리오에 바로 연결할 수 있다.
  - 사람과 의료 장비가 섞인 환경에서 안내 멘트와 대기/우회 판단 근거로 활용할 수 있다.
